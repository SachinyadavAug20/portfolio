- it is from `java.lang` package
- it is parent of all class.Every class in java inherit form object class(internally, directly or indirectly). if extent some class then it will indirectly extend it by extending it's parent(because multiple inheritance not possible)
Reason for `Object` class existence
- common behavior which needed in all class
- it store reference of any class(as it parent) => polymorphous
there are 4 core methods in `Object` class
- Core methods
	- `toString()`
	- `equals()`
	- `hashCode()`
	- `getClass()`
- Cloning
	- `clone()`
- Garbage collection
	- `finalize()`  // depricated
- Threads
	- `wait()`
	- `notify()`
	- `notifyAll()`
## `toString()`
convert anything to string. => return string representation of that object
This is can be `@override` to adjust to the user-defined class
by using default implementation it give -> `ClassName@HexCode` Hex-code is unique code for this class
```java
public class demo {
    public static void main(String[] args) {
        Student s=new Student(102, "sachin");
        System.out.println(s.toString()); // Student@1dbd16a6   // default toString
        System.out.println(s.toString()); // Student [rollno=102, name=sachin]     
    }
}
class Student {
    int rollno;
    String name;
    public Student(int rollno, String name) {
        this.rollno = rollno;
        this.name = name;
    }
    @Override
    public String toString() {
        return "Student [rollno=" + rollno + ", name=" + name + "]";
    }
}
```
## `equals()`
It compares two object return true or false
```java
class Object{
	public boolean equals(Object o){
		return o==this;
	}
}
```
default compares reference (this won't check value to be equal it only check reference(check if shallow copy))
```java
public class demo {
    public static void main(String[] args) {
        Something s1 = new Something(12);
        Something s2 = new Something(12);
        // System.out.println(s1.equals(s2)); // false -> by default implementation
        System.out.println(s1.equals(s2)); // true -> by new implementation
    }
}
class Something {
    int ss;
    Something(int ss){
        this.ss = ss;
    }
    @Override
    public boolean equals(Object o){
        Something s = (Something)o;
        return this.ss==s.ss;
    }
}
```
can have `nullPointerException` and can have `classTypeCastError` 
```java
public class demo {
    public static void main(String[] args) {
        Something s1 = new Something(12);
        Something s2 = new Something(12);
        // System.out.println(s1.equals(s2)); // false -> by default implementation
        System.out.println(s1.equals(s2)); // true -> by new implementation

        Something s3 = null;
        // System.out.println(s1.equals(s3)); // main" java.lang.NullPointerException: Cannot read field "ss" because "<local2>" is null
        System.out.println(s1.equals(s3)); // false
    }
}
class Something {
    int ss;
    Something(int ss){
        this.ss = ss;
    }
    @Override
    public boolean equals(Object o){
	    if(this==o) return true;
        if(o==null || !(o instanceof Something)) return false;
        Something s = (Something)o;
        return this.ss==s.ss;
    }
}
```
## `hashCode()`
This return a hash-code which is unique for that object.This are hex-decimal code.
```java
class Object{
	public int hashCode(){
		// default implementation 
		return 0x...;
	}
}
```
Rule in java 
- if two objects are equal then there hash-code must be equal.converse might not be true(depending on implementation of `equals()`)
- if this rule breaks it will give `exception` in `hashset`,`hashmap`,`hashtable` implementation
This rule will be true if not override the default `equals()`, thus while overriding `equals()` be carefully ==> if override one override both
```java
public class demo {
    public static void main(String[] args) {
        Something s1 = new Something(12);
        Something s3 = new Something(12);
        System.out.println(s1.equals(s3)); // true
        // PROBLEM
        System.out.println(s1.hashCode()==s3.hashCode()); // false
    }
}
class Something {
    int ss;
    Something(int ss){
        this.ss = ss;
    }
    @Override
    public boolean equals(Object o){
        if(o==null || !(o instanceof Something)) return false;
        Something s = (Something)o;
        return this.ss==s.ss;
    }

    @Override
    public int hashCode(){
        return ss;
    }
}
```
solve by overriding the hash.
```java
public class demo {
    public static void main(String[] args) {
        Something s1 = new Something(12);
        Something s3 = new Something(12);
        System.out.println(s1.equals(s3)); // true
        System.out.println(s1.hashCode()==s3.hashCode()); // true
    }
}
class Something {
    int ss;
    Something(int ss){
        this.ss = ss;
    }
    @Override
    public boolean equals(Object o){
        if(o==null || !(o instanceof Something)) return false;
        Something s = (Something)o;
        return this.ss==s.ss;
    }

    @Override
    public int hashCode(){
        // prime number -> for unique
        int prime = 17;
        int result = prime*31+this.ss;
        return result;
    }
}
```
by using primes chances of unique number increases
to avoid `nullPointerException` 
```java

    }
```

## `getClass()`
It returns runtime class of an object.
```java
import java.util.Objects;

public class demo {
    public static void main(String[] args) {
        Something s1 = new Something(12);
        Object s2 = new Something(12);
        System.out.println(s1.getClass()); // class Something
        System.out.println(s2.getClass()); // class Something
    }
}
class Something {
    String name;
    int age;
    Something(int age) {
        this.age = age;
    }
}
```
`getClass()` is a final method can't override. points
- it return a object of type `Class`
- can get class name by 
```java
        System.out.println(s1.getClass().getName()); // Something
        System.out.println(s2.getClass().getName()); // Something
```
## `instanceof`
it is a operator which gives say weather a object is instance of class/sub-class
```java
        System.out.println(s1.getClass().getName()); // Something
        System.out.println(s2.getClass().getName()); // Something
        System.out.println(s1 instanceof Something); // true
        System.out.println(s1 instanceof Object);    // true
```
example
```java
public class demo {
    public static void main(String[] args) {
        Animal a1 = new Animal();
        Animal a2 = new Dog();
        System.out.println(a1.getClass().getName()); // Animal
        System.out.println(a2.getClass().getName()); // Dog
        System.out.println(a1 instanceof Dog);       // false
        System.out.println(a2 instanceof Dog);       // true
        System.out.println(a1 instanceof Animal);    // true
        System.out.println(a2 instanceof Animal);    // true
    }
}

class Animal{

}
class Dog extends Animal{
}
```
## `Clone()`
It creates a copy of the object. and it is protected method
- this always make a shallow copy of the object.(it is same as doing the reference assignment)
```java
class Object{
	protected Object clone() throw classNotSupportedException{
		
	}
}
```
can override `Clone()` if and only if `implements Cloneable` interface ==> gives error
```java
public class demo {
    public static void main(String[] args) throws CloneNotSupportedException {
        Dog a = new Dog("dog",5);
        Dog b = (Dog)a.clone(); // java.lang.CloneNotSupportedException: Dog
    }
}

class Dog{
    String name;
    int speed;
    Dog(String name,int speed){
        this.name = name;
        this.speed = speed;
    }
    @Override
    protected Object clone() throws CloneNotSupportedException{
        return super.clone();
    }
}
```
solve error 
```java
public class demo {
    public static void main(String[] args) throws CloneNotSupportedException {
        Dog a = new Dog("dog",5);
        Dog b = (Dog)a.clone(); 
    }
}

class Dog implements Cloneable{
    String name;
    int speed;
    Dog(String name,int speed){
        this.name = name;
        this.speed = speed;
    }
    @Override
    protected Object clone() throws CloneNotSupportedException{
        return super.clone();
    }
}
```
because
- not every object should be clone-able
- need to tell java to make it clone-able object by `implements Cloneable`
- This interface `Cloneable` is empty thus there is nothing to override ==> called marker interfaces.
