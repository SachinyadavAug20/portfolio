## Auto boxing /Unboxing
###### java give class for primitive data types with different name,
- primitive -> wrapper
- int -> integer
- long -> Long
- short -> Short
- float -> Float
- double -> Double
- char -> Char
This wrapper is for non-primitive heap class -> reason for it
- primitive exist for legacy(C and C++ has it)
- primitive is fast and low level
- collection frame works need this wrapper (like for STL in C++)
direct convert primitive to wrapper called auto-boxing
- value assignment
- method class 
- in arithmetic operation (internally uses `.intValue`) 
```tsx
int x=10;
Integer y=x;  // auto boxing 
// at run time compiler 
// Integer y=new Intger(x); //old java way
// Integer y=Integer.valueOf(x); // new java caching optimize -> valueOf is a static method of Integer class
```
###### unboxing opposite of auto-boxing.
```java
Integer x=10;
int y=x; // unboxing
// internally it does this
// int y=x.intValue();
// intValue is a non-static method in Integer class(getter) 
```
example 
```java
public class demo {
  public static void main(String[] args) {
      int x=10;
      Integer y=x; // auto-boxing

      Integer a=new Integer(20);
      int b=a;    // unboxing

	  Integer c=22;
      System.out.println(y); // 10  unboxing
      System.out.println(b); // 20
      System.out.println(c); // 22
  }
}
```
###### can get null pointer exception using wrapper during unboxing
```java
public class demo {
  public static void main(String[] args) {
      Integer a=null;
      int x=a;
      System.out.println(x); // java.lang.NullPointerException: Cannot invoke "java.lang.Integer.intValue()" because "<local1>" is null
  }
}
```
###### during comparison don't unboxing happens automatically. it tries to compare reference which always false(like C/C++ string comparison issue)
```java
public class demo {
  public static void main(String[] args) {
      int x=10;
      int y=10;
      System.out.println(x==y);// true

      Integer a=1110;
      Integer b=1110;
      System.out.println(a==b); // false
      System.out.println(a.equals(b)); //true
  }
}
```
###### compiler does many things internally so wrapper around the primitives behaviors like primitive itself
###### caching used which is used and is more optimize -> Integer cached -128 to 127 object as mostly used ->thus, if make `x=100` and `y=100` it references to same cached object for 100 thus, give true
```java
      Integer a=10;
      Integer b=10;
      System.out.println(a==b); // true
      System.out.println(a.equals(b)); //true
```
## Abstract classes
###### class which can't be instanced. points
- cannot be instantiated directly (need to use inheritance)
- can contain abstract method(method without implementations)
- can also contain normal methods
- is meant to be extended
###### question pointers
- abstract class can have constructor for use in child constructor
```java
public class demo {
  public static void main(String[] args) {
      Animal a=new dog("QWERTY");
      a.makeSound();  // barking
  }
}

abstract class Animal{
    string name;
    abstract void makeSound();
    Animal(string n){
        name=n;
    }
    void sleep(){
        System.out.println("sleeping");
    }
}
class dog extends Animal{
    @Override
    Dog(string n){
        super(n);
    }
    void makeSound(){
        System.out.println("barking");
    }
}
```
- can't make abstract class final as final make it un-inheritable and abstract class are mean to be inherited -> **a class can be abstract or final not both**
- abstract class can have static method and variable as it is linked to class
```java
public class demo {
  public static void main(String[] args) {
      Animal a=new dog("Tommy");
      a.makeSound();                   // barking
      System.out.println(Animal.type); // mammal
  }
}

abstract class Animal{ 
    String name;
    static String type="mammal";
    abstract void makeSound();
    Animal(String n){
        name=n;
    }
    void sleep(){
        System.out.println("sleeping");
    }
}
class dog extends Animal{
    dog(String n){
        super(n);
    }
    @Override
    void makeSound(){
        System.out.println("barking");
    }
}
```
- can make abstract method private but it should be defined not abstract -> as child class can't see it and thus can't define it.
- can make final method in abstract class -> which are not abstract `as final method can't be override by child`
- can make abstract class with no abstract method because if need to make class whose object can't be made -> like give default implementation for all methods
