## JVM (Java virtual machine)
###### every-platform has it's own JVM which run the BYTE-CODE of java(which is output of compiler)
###### It translate BYTE-CODE to machine code
###### Compiler(convert full file once) and interpreter(convert line by line).

> [!Note]
> Java is compiled(make BYTE-CODE) and interpreted(by JVM) language

###### JVM as interpreter is old(19s).Now, JIT(just in time) compiler is added to JVM.==> Thus, now JVM has compiler and interpreter
###### JVM --> convert frequently used code using JIT and use interpreter for the rest of the things(compilation slow and run comparable to C and C++)
###### JVM also provide security by using sand-box model
###### JVM has garbage collector

## JRE (Java runtime environment)
###### JRE = JVM + class libraries to run java program properly
###### Internal libraries for tasks like console I/O, file etc for all imports 
###### Won't get separate JVM or JRE need to get JDK
## JDK (Java development kit)
###### It has everything to write and run java code 
###### JDK = JRE + compiler,Debugger + Docs etc
