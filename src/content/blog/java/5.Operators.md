## Arithmetic operator
###### + - * / % are basic 5 operator
###### += and ++ also work like in C/C++
```java
    // arithematic operator
    int a = 10, b = 20;
    int sum = a + b;
```

## Bitwise Operator
###### & | ^ ~ >> << <<= >>= &= are similar to in C/C++
###### >>> <<< this are new shift with zero fill
###### b << n === b * 2^n
###### b >> n === b /2^n
following binary representation rule of representation of that datatype
###### bitwise operators are only applicable on int and long(will auto type promote)
```java
    byte b1=2,b2=1;
    // byte ans=b1>>b2; // both bytes still ERROR BUZ butwise only works in int and long
    byte ans=(byte)(b1<<b2);
    System.out.println(ans); // 8
```

```java
    // Bitwise operator
    byte b = 5;
    System.out.println(~b); // ~00000101 = 111111010 = -6
    System.out.println(5 & 3); // 101 & 011 = 001 === 1
    System.out.println(5 | 3); // 101 | 011 = 111 === 7
    System.out.println(5 ^ 3); // 101 ^ 011 = 110 === 6
    System.out.println(10 << 2); // 1010 << 2 = 101000 === 40
    System.out.println(1<<2); // 1 << 2 = 100 == 4
    System.out.println(1024>>2); // 10000000000 >> 2 = 100000000 = 256
    System.out.println(1>>1); // 1 >> 1 = 0 === 0
    
    // typecasting
    byte a = 1;
    // a=a<<4; // error
    a=(byte)(a<<4); 
    System.out.println(a); // 16 
```

###### It(>>>) has specific use cases 
```JS
<<   left shift
>>   signed right shift
>>>  unsigned right shift
```
example
```java
        byte b = -128;

        System.out.println("Original:");
        System.out.println(Integer.toBinaryString(b));

        System.out.println("\nLeft Shift <<");
        int left = b << 1;

        System.out.println(left);
        System.out.println(Integer.toBinaryString(left));

        System.out.println("\nSigned Right Shift >>");
        int right1 = b >> 1;

        System.out.println(right1);
        System.out.println(Integer.toBinaryString(right1));

        System.out.println("\nUnsigned Right Shift >>>");
        int right2 = b >>> 1;

        System.out.println(right2);
        System.out.println(Integer.toBinaryString(right2));}
```

## Logical operator
###### &&(and) ||(or) --> this works on expression
```java
    // logical operators
    System.out.println(10 < 20 && 10 > 2); // true
    System.out.println(10 < 20 || 10 > 2); // true
```
###### short circuit -> in && order matters and short circuit
```java
// example :
if(indexValide(i) && arr[i]==11){
	return true;
}
```
here, if index is invalid then `arr[i]` will give error so, when index invalid -> false it will not check `arr[i]` directly becomes false
thus, it make order important if `arr[i]==11 && indexValide(i)` will give error every time when invalid index is reached
###### short-circuit work in &&(if 1st false not check 2nd => return false) and ||(if 1st true not check 2nd => return true)
#### how to avoid short-circuit
###### help we get by boolean ==> &&=& and ||= | because it does single bit bitwise on boolean --> but this will not do short-circuit
```java
    // logical operators without short-circuit(buz it is bitwise operator)
    System.out.println(10 < 20 & 10 > 2); // true
    System.out.println(10 < 20 | 10 > 2); // true
```

## Precedence
###### Like BODMAS for code. Rules

| Precedence | Operators                                                       | Description                              | Associativity |
| ---------- | --------------------------------------------------------------- | ---------------------------------------- | ------------- |
| 1(highest) | `()` `[]` `.`                                                   | Parentheses, array access, member access | Left → Right  |
| 2          | `expr++` `expr--`                                               | Postfix increment/decrement              | Left → Right  |
| 3          | `++expr` `--expr` `+` `-` `!` `~` `(type)`                      | Unary operators                          | Right → Left  |
| 4          | `*` `/` `%`                                                     | Multiplication, division, modulo         | Left → Right  |
| 5          | `+` `-`                                                         | Addition, subtraction                    | Left → Right  |
| 6          | `<<` `>>` `>>>`                                                 | Bitwise shift operators                  | Left → Right  |
| 7          | `<` `<=` `>` `>=` `instanceof`                                  | Relational operators                     | Left → Right  |
| 8          | `==` `!=`                                                       | Equality operators                       | Left → Right  |
| 9          | `&`                                                             | Bitwise AND                              | Left → Right  |
| 10         | `^`                                                             | Bitwise XOR                              | Left → Right  |
| 11         | `\|`                                                            | Bitwise OR                               | Left → Right  |
| 12         | `&&`                                                            | Logical AND                              | Left → Right  |
| 13         | `\|`                                                            | Logical OR                               | Left → Right  |
| 14         | `?:`                                                            | Ternary operator                         | Right → Left  |
| 15(lowest) | `=` `+=` `-=` `*=` `/=` `%=` `&=` `^=` `\|=` `<<=` `>>=` `>>>=` | Assignment operators                     | Right → Left  |
###### Always use as much () as possible for code-readability
