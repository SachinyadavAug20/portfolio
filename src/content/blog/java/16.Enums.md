## enums
enums: Enumerations (enumerated types) is a predefined set of constants(convection to write name of constant with all caps)
```java
public class demo {
    public static void main(String[] args) {
        enum paymentStatus{
            SUCCESS,FAILED,PENDING;
        }
        paymentStatus ps=paymentStatus.SUCCESS;
        System.out.println(ps.name());  // SUCCESS
        if(ps==paymentStatus.SUCCESS){  // payment successfully
            System.out.println("payment successfully");
        }
    }
}
```
now have
- type safety -> can chose any one from enums
- enum is a class internally in java.This special class extends the `Enum` class from `java.lang.Enum` package
- each constant are by default are `static` and `final`. and are objects of of this enum class
- and makes a `private constructor` so it is not possible to make it's object outside the class.(for safety can't)
```java
// made enum Direction
final class Direction extends Enum<Direction>{
	public static final Direction NORTH=new Direction();
	public static final Direction SOUTH=new Direction();
	public static final Direction EAST=new Direction();
	public static final Direction WEST=new Direction();
	private Direction(){
		
	}
}
```
- in heap memory each constant object is present.and allocate by using class name`Direction.NORTH`. can do `Direction d=Direction.NORTH`.
- as points to same reference thus will be unique and comparable.(act as marker)
example of Day of the week:
```java
public class demo {
    public static void main(String[] args) {
        enum Day {
            MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY

        System.out.println(today);        // SUNDAY
        System.out.println(today.name()); // SUNDAY
    }
}
```

What are existing methods in enum from `Enum` class
- `values()`
- `valueOf(string)`
- `name()`
- `ordinal()`

> [!Note]
> `name` and `ordinal` are there in `Enum` class and `values` and `valueOf` is made by compiler 
## `values()`
it give array of all values in enum
```java
public class demo {
    public static void main(String[] args) {
        for(Direction d:Direction.values()){
            System.out.println(d);
        }
        /*
UP
DOWN
LEFT
RIGHT
         */
    }
}
enum Direction{
    UP,DOWN,LEFT,RIGHT
}
```
## `valueOf()`
convert a string into a enum constant
```java
public class demo {
    public static void main(String[] args) {
        Direction d=Direction.valueOf("UP");
        System.out.println(d);  // UP
    }
}
enum Direction{
    UP,DOWN,LEFT,RIGHT
}
```
should be exactly same case.
## `name()`
exact name of object
can't override `name`. but can override `toString` to get specific message
```java
public class demo {
    public static void main(String[] args) {
        Direction d=Direction.valueOf("UP");
        System.out.println(d);              // UP
        System.out.println(d.name());       // UP
        System.out.println(d.toString());   // UP
    }
}
enum Direction{
    UP,DOWN,LEFT,RIGHT
}
```

###### why `values()` and `valueOf()` are made by compiler ?
- compiler handles constant object so it has to make this methods
- it makes array of `$values` which it returns on `values()` method => made it static and return clone of the array of references => return clone so deep copy can't override it.
```java
final class Direction extends Enum<Direction> {
	...
    private static final Direction[] $VALUES = {
        NORTH,
        SOUTH,
        EAST,
        WEST,
    };

    public static Direction[] values() {
        return $VALUES.clone();
    }
    public static Direction valueOf(String s) {
	    return super.valueOf(Direction.class, s);
	}
    ...
}
```
- `valueOf()` is also in `Enum` class but it is override by the compiler
