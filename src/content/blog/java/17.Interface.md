## interface
it is list of method which are abstract => it tell what is to be done without implementation
Interfaces defines what an object can do without telling how it does that
example
```java
interface car{
    public void run();
    public void drive();
}
```
 interface is a blueprint of behavior. => it is a contract to define all methods declared in it.
 interface all methods are by default public so, while implementing it need to make it `public abstract`
 ```java
 public class demo {
    public static void main(String[] args) {
        Car c=new BlackThar();
        c.drive(); // Thar is driving
        c.run();   // Thar is running
    }
}
interface Car{
     void run();
     void drive();
}
abstract class Thar implements Car{
    abstract public void run();
    abstract public void drive();
}
class BlackThar extends Thar{
    @Override
    public void run(){
        System.out.println("Thar is running");
    }
    @Override
    public void drive(){
        System.out.println("Thar is driving");
    }
}
 ```
Interface uses dynamic polymorphic(dynamic dispatch)
```java
public class demo {
    public static void main(String[] args) {
        Payment p1=new DebitCard();
        Payment p2=new CreditCard();
        p1.play(); // Pay with CreditCard
        p2.play(); // Pay with DebitCard
    }
}

interface Payment {
    void play();
}
class CreditCard implements Payment {
    @Override
    public void play() {
        System.out.println("Pay with CreditCard");
    }
}
class DebitCard implements Payment {
    @Override
    public void play() {
        System.out.println("Pay with DebitCard");
    }
}
```
it is base for `sprintboot framework` it leads to clean code
## Variables inside `interfaces`
by default every variable is `public static final` because it can't link variable with `interface` 
```java
interface MathConstant{
    double PI_VALUE=3.14;
}
```
compiler does
```java
interface MathConstant{
    public static final double PI_VALUE=3.14;
}
```
used as
```java
public class demo {
    public static void main(String[] args) {
        MathConstant r1=new Random();
        // r1.f(); // Error as MathConstant don't have f()
        Random r2=new Random();
        r2.f();                                    // 3.14
        System.out.println(MathConstant.PI_VALUE); // 3.14
    }
}

interface MathConstant{
    double PI_VALUE=3.14;
}
class Random implements MathConstant{
    void f(){
        System.out.println(PI_VALUE);
    }
}
```
thus can use it as constant container to store constants
## multiple inheritance supported by interfaces
```java

	}
```

```java
    void eat();
```

## After java-8 
interface can have default methods ==> makes this methods optional to override
```java
interface Animal{
    default void eat(){
        System.out.println("Animal Eating...");
    }
}
interface Dog extends Animal{
    default void bark(){
        System.out.println("Dog Barking...");
    }
}
class StreetDog implements Dog{
    @Override
    public void eat() {
        System.out.println("Eating...");
    }
    // optional overrides
    @Override
    public void bark() {
        System.out.println("Barking...");
    }
}
```
why need to have default method ==> java has `list interface` need to have more methods in it. but to avoid breaking need to give default implementation. i.e to have flexibility.
also added static method to => which automatically makes it default(can call by using interface name)
```java
interface Animal{
    default void eat(){
        System.out.println("Animal Eating...");
    }
    static void hello() {
        System.out.println("Hello");
    }
}
interface Dog extends Animal{
    default void bark(){
	    Animal.hello();
        System.out.println("Dog Barking...");
    }
}
class StreetDog implements Dog{
    @Override
    public void eat() {
	    Animal.hello();
        System.out.println("Eating...");
    }
    @Override
    public void bark() {
	    Animal.hello();
        System.out.println("Barking...");
    }
}
```

## Marker interface
no method 
```java
interface A{}
```
to mark something is doable e.g(real): `Cloneable`,`Serializable`,`RandomAccess`
example: `Object` class has `clone()` function any class if want to override this `clone()` should implement `cloneable interface` which act as a marker for compiler. (to avoid cloning of secure/critical data)
