Problem: in a program we have many string so if for each we need a object this will lead to memory issue ?
- when string declared using literal `String s="Hello";`. this string is stored in at a place in heap called as string pool(this tries to reuse string so don't have to make many string (if same string they won't make different object they will point to same reference)) 
- thus in java 2 string of same value have same reference thus can do `s1==s2` to compare string directly
```java
class demo{
    public static void main(String[] args) {
        String s1="meow";
        String s2="meow";
        String s3=new String("meow");
        System.out.println(s3);     // meow
        System.out.println(s1==s2); // true ==> because of string pool
        System.out.println(s2==s3); // false  => because of new object in heap outside pool
    }
}
```
Now, in modern java String are stored in string pool which is in string pool(previously it was going special memory `premGen`)
String class has override the `equals()` so it compare value 
```java
class demo{
    public static void main(String[] args) {
        String s1="meow";
        String s2="meow";
        String s3=new String("meow");
        System.out.println(s3);     // meow
        System.out.println(s1==s2); // true ==> because of string pool
        System.out.println(s2==s3); // false  => because of new object in heap outside pool
        System.out.println(s2.equals(s3)); // true
    }
}
```
for which string goes to string pool there is a rule

> [!Golden rule]
> Only compile time constant go to string pool automatically.
> Runtime(using `new`) created string go to heap.

examples
```java
class demo{
    public static void main(String[] args) {
        // 1
        String s1="Ja"+"va";
        String s2="Java";
        System.out.println(s1==s2); // true => as all are compile time constant

        // 2
        String s3="ja";
        String s4=s3+"va";  // here s4 is not constant at compile time -> not in string pool
        s3+="va";
        System.out.println(s3==s4); // false
        System.out.println(s3.equals(s4)); // true

        // 3
        String s5="java";
        String s6=s5;
        System.out.println(s5==s6); // true => direct pass by reference -> compiler time resolved

        // 4
        String s7="java";
        s7="javascript";
        String s8="javascript";
        System.out.println(s7==s8); // true

        // 5
        String s9=new String("hello");
        // here "hello" goes to string pool and then, s9 is make in heap with same value as "hello" #ruleAllLiteralsGoToPool
    }
}
```
Note: all literals(stored reference or not) will go to string pool

## String class
```java
Public final class String{
	private final byte[] value; // to save memory 
	private final byte coder;
	private final int hash; // final as string immutable
	
}
```
array of bytes is a array of Unicode value.(instead by char(before java 9) used byte)
- as mostly there are ASCII values in string (but char takes 2 bytes as Unicode) thus can use a byte instead of full char ==> saved 50% memory
- if need Unicode character => has a marker coder -> 0,1(to tell which format string is in)
- when coder == 0 : all values are ASCII (`Latin1` format)
	- read only 1 byte to get character
- when coder == 1 : need full Unicode (`UTF-16` format)
	- read 2 bytes to get character
- in bytes when need to store Unicode --> will store 2 byte for each character and make the coder=1
Hash is a hex-code --> unique for a string==> it is stored as it will not change(!immutable) So, no need to compute every-time(cached)
#### String optimization
- String pool
- `char[]` -> `byte[]`
- cached the hash value

- `lastIndexOf()` -> similar to `indexOf()` but give last instance index

- `substring(beginIndex,endIndex)` -> get sub-string in range mathematically it gives `[beginIndex,endIndex)`

- `toLowerCase()`

It is mutable make longer string buffer => have extra space to be appended
Thus, have count(occupied place) and capacity of array(in start it is 16 -> 34 -> 70 ...) when capacity filled it doubles--> (`new capacity = 2 * initial +2`)
similar used in String buffer
can also give capacity
- `new StringBuilder()` make empty string(default 16 capacity)
- `new StringBuilder(str)` make it of that string + extra capacity of 16 (e.g "java" then capacity is 20)

### Method of String Builder and buffer
#### StringBuilder / StringBuffer
- `append()`  
- `insert()`
- `delete()`
- `replace()`
- `reverse()`
- `charAt()`
- `setCharAt()`
- `length()`
- `capacity()`
- `ensureCapacity()`
- `trimToSize()`
```java
class demo {
    public static void main(String[] args) {
        StringBuilder sb=new StringBuilder();

        sb.append("Hello");
        System.out.println(sb); // Hello
        
        sb.append("World");
        System.out.println(sb); // HelloWorld

        sb.insert(5, " ");
        System.out.println(sb); // Hello World

        sb.delete(5,11);
        System.out.println(sb); // Hello

        sb.deleteCharAt(4);
        System.out.println(sb); // Hell

        sb.replace(0, 4, "Hi");
        System.out.println(sb); // Hi

        sb.reverse();
        System.out.println(sb); // iH

        System.out.println(sb.charAt(1));

        sb.setCharAt(1, 'l');
        System.out.println(sb); // il
        System.out.println(sb.length()); // 2
        System.out.println(sb.capacity()); // 16

        sb.append("123456789012345");
        System.out.println(sb.length());  // 17
        System.out.println(sb.capacity());// 34

        sb.ensureCapacity(50); // ask for more space
        System.out.println(sb.capacity()); // 70

        sb.trimToSize();
        System.out.println(sb.length());   // 17
        System.out.println(sb.capacity()); // 17
    }
}
```

