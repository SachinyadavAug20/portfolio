###### We have data types integer(byte,short,int,long), real(double,float), character(char) and boolean
###### as all of them are binary internally thus can convert from one type to other if possible
###### There are two types of type conversions in Java
- Implicit(automatically)
- Explicit(need to specify)
## Implicit conversion (widening conversion)
```Java
    //Implicit type conversion
    byte b = 10;
    int i = b; // implicit
    System.out.println(i); // 10

```
###### Rules 
- Destination data types must be wider than source data type(byte -> int possible but int -> byte not)
```Java
    //Implicit type conversion
    byte b = 10;
    int i = b; // implicit
    byte b2=i;
    System.out.println(i+b2); // error: incompatible types: possible lossy conversion from int to byte
```
###### character can be converted to int (char = 16 bit and int 32 bit) thus can convert
```Java
    // Implicit
    char c = 'a';
    int i=c;
    System.out.println(i); // 97
    
    // Explicit
    char kk='k';
    byte b=(byte)(int)kk; // !unnessary int casted
    System.out.println(b); // 107
    int x=97;
    char y=(char)x;
    System.out.println(y); // a
```
## Explicit conversion(casting)
###### for narrowing size use typecasting
```Java
    //explicit type conversion
    byte b = 10;
    int i = b; // implicit
    byte b2=(byte)i; // explicit
    System.out.println(i+b2); // 20 
```
###### truncation problem when shrink size
```Java
    //explicit type conversion
    int i=300; // 000000000000000100101100 32 bits is 300
    byte b=(byte)i; // 00101100 8 bit is 300-2^8 == 44 
				    //OR b=i%Range(byte) = 300 % 128 = 44
    System.out.println(b); // cycle back > 44
```
###### long -> int, double -> float is called narrowing conversion
## Truncating conversion
###### float -> int (same range)
```Java
    float f=3.12f;
    int i=(int)f;
    System.out.println(i); // 3
```
###### It is not rounding off => it ignores to decimal values not rounding
## Boolean to any data types
###### not possible
```java
    boolean b = true;
    char c=(char)b;
    System.out.println(c); // error: incompatible types: boolean cannot be converted to char
    int i=(int)b;
    System.out.println(i); // error: incompatible types: boolean cannot be converted to int
```
###### boolean is special takes only 1 bit.
## Defaults
###### any literal numbers is always -> int
###### any literal real number is -> double
```Java
    byte b = 10;
    b=b*3; // error: incompatible types: possible lossy conversion from int to byte
    System.out.println(b);
```
correct way
```java
    byte b = 10;
    b=(byte)(b*3);
    System.out.println(b); // 30
```
## Automatic type promotion
###### operations are done only on int and then it is transferred to data type using rule ![[Pasted image 20260514172959.png|327]] 
```java
    byte b = 10;
    int i=29;
    b=(byte)(b*i);  // byte * int = int so need to convert
    System.out.println(b); // 34 == 290 % 128

```
###### Rules of type promotion
- byte, short and char values are promoted to int in any operation
- when doing operation between data of different data types result is promoted to bigger of the operand datatype
