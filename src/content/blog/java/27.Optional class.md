This is to solve null-pointer-exception problem(things can return null)
we do check if (null) => can't check for everything(like a user class -> has address object -> has city => need to do many null check at all steps) => null check hell
#### Problem with null
- in general null means nothing,but in particular it has no meaning(error,empty,no exist...)
- can't call any method on null but, it can be stored in in any data type which methods are callable(thus, gives null pointer exception)
To solve all this issues have a Optional class
### Optional class
It is a wrapper for any object `Optional<String>` it may or may not have a value.
`name.ifPresent()` if there is something, to consumer lambda
it is a good practice to return optional if expected to have null
```java
Optional<String> name=Optional.of("MeowMegaCorp");
```
internally optional class is 
```java
class Optional<T>{
	private final T value;
}
```
call if have idea that need to write safe code
Methods of optional
- `Optional.of("QWERTY")` -> can't pass null this way
- `Optional.ofNullable(somethingMayBe)` -> can pass value or null
- `Optional.empty()` -> object is null only
- `o.isPresent(Consumer interator)` consumes the value
#### Getting value
- `name.get()` -> give value if present and gives `noSuchValueException` when it is null (not to use directly)
- `name.isPresent()` -> check is present can use `.get()` with it for being safe
- `o.ifPresent(consumer)` -> 
- `o.orElse(value)` -> if have null then add this value(not change original value if present)
- `o.orElseGet(value)` 
- `o.orElseThrow()` -> if null then throw exception
- `o.ifPresentOrElse(consumer)` -> 
```java
import java.util.*;

public class demo {
    public static void main(String[] args) {
        Optional<String> name = getName();
        if(name.isPresent()){
            System.out.println(name.get()); // Meow
        }
        name.ifPresent(System.out::println); // Meow

        Optional<String> name2 = Optional.empty();
        System.out.println(name2.orElse("No name")); // No name

        Optional<String> name3 = Optional.ofNullable("Meow");
        // it allows shortckt
        System.out.println(name3.orElseGet(() -> "No name")); // Meow
        // it evaluate it no matter
        System.err.println(name3.orElse("No name")); // Meow

        Optional<String> name4 = Optional.ofNullable(null);
        // System.out.println(name4.orElseThrow(() -> new RuntimeException("No name")));

        Optional<Integer> age = Optional.ofNullable(null);
        age.ifPresentOrElse(System.out::println,() -> System.out.println("No name")); // No name

    }
    public static Optional<String> getName(){
        return Optional.of("Meow");
    }
}
```

public class demo {
    public static void main(String[] args) {
        Optional<String> name = Optional.of("Sachin");
        Optional<String> result1 = name.filter(s -> s.startsWith("S"));
        Optional<String> result2 = name.filter(s -> s.startsWith("A"));

        System.out.println(result2.orElse("Not found")); // Not found
    }
}

#### Optional principle
It is not a replacement for null
- if a method have to return data which may be null Use optional
- use it as return type, not good practice to use it as a datatype

| Stream             | Optional        |
| ------------------ | --------------- |
| 0 or many elements | 0 or 1 elemenet |
If combine -> handle maybe values inside multiple values
for every user -> check if have email(returns optional of string)
```java
user.stream()
	.map(u->u.getEmail()) // stream of optional<String>
	.filter(x->x.isPresent()) // apply a function on optional
	.map(x->x.get()) // no null exception
```
