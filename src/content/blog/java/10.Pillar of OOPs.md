## Encapsulation
###### real word object have encapsulation as a property
###### Encapsulation: to form capsule of data and behavior
- Both data & behavior should be together within an object
- we should not provide unrestricted access to our data -> provide restricted getter and setter methods
```java
public class animal {
  int noOfLegs;
  String name;
  void move(){
    System.out.println("I am moving");
  }
  void eat(){
    System.out.println("I am eating");
  }
}
```

- protected (in same package + inherited class)

## Abstraction
###### Abstraction: The process of focusing on what something does, while ignoring how it does that.
###### we don't learn to make car how it works -> we only know what will it do.
###### ATM == deposit,withdraw,check balance
###### ATM != circuit,electronic,metal,welding etc
###### Oops : mi-mikes persecution of real world 
- represent what ever necessary
- even what we model, don't want everyone to know how everything works,but they can still use it. (give interface)
###### 2 types of abstraction
- low level (hiding implementation details)
- high level (separate WHAT from HOW)
###### main and car are in different files. not opened it and don't know how it is implemented -> low level abstraction
###### high level -> using abstract class and interface
```java
public class demo {
  public static void main(String[] args) {
    car c = new car(); // tightly compuled
    c.setSpeed(100);
    System.out.println(c.getSpeed());
  }
}
class car { // concrete class
  private int speed = 0;
  public void setSpeed(int speed) {
    this.speed = speed;
  }
  public int getSpeed() {
    return this.speed;
  }
}
```
###### electric and fuel car have same method but different method -> what and how is coupled --> low level abstraction
###### use inheritance -> both fuel and electric inherit car parent car --> thus hidden details

## Interface
###### Interface are similar to abstract class -> it has nothing to do with object
###### Interface is a contract -> can define and declare default function
```java
interface car{
  void start();
  void accelarate();
  void brake();
}
class fuelCar implements car{
  @Override
  public void start() {
    System.out.println("start");
  }
  void accelarate(){
    System.out.println("accelarate");
  }
  void brake(){
    System.out.println("brake");
  }
}
class electricCar implements car{
  @Override
  public void start() {
    System.out.println("start");
  }
  void accelarate(){
    System.out.println("accelarate");
  }
  void brake(){
    System.out.println("brake");
  }
}
```
###### `interface` is pure "WHAT" -> pure abstraction

| interface                                                             | abstract class                            |
| --------------------------------------------------------------------- | ----------------------------------------- |
| pure WHAT ?(pure abstraction)                                         | partial WHAT & HOW                        |
| like contract for class what to implememt                             | no such intentation show for class        |
| responsibilities / capabilities / role for a class are cleared by it. | it shows families of similar object       |
| name as "ICompar**able**", "IFly**able**","IWalk**able**"             | names as "Car","Human","Vehical","People" |

## Polymorphism
###### many form -> Polymorphism
###### humans show different forms -> like human walks different when something changes
###### same object behaves/response differently on same command if one parameter is changed.
#### Types of Polymorphism
###### compile-time Polymorphism / using method overloading
```java
class Human{
  void run(){
    System.out.println("run");
  }
  void run(boolean isDogBehild){
    if(isDogBehild){
      System.out.println("walk");
    }
  }
}
```
###### Run-time/Dynamic Polymorphism using method overriding
```java
public class demo {
  public static void main(String[] args) {
  // decide on runtime/dynamic
      Animal human = new Human();
      human.run();  // walking
      Animal duck = new Duck();
      duck.run();   // swim
      Animal dog = new Dog();
      dog.run();   // run
  }
}

abstract class Animal{
  abstract void run();
}
class Human extends Animal{
  void run(){

  }

class Duck extends Animal{
    void run(){
      System.out.println("swim");
    }

```
