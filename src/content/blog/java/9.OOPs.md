> [!Key rule]
> A non-static inner class belongs to an instance of the outer class, so you can't do new InnerClass() inside a static method (like main). Fix: add static to the inner class — static class Student { — to make it a static nested class. That's the right call anyway when the inner class doesn't access outer-class instance fields.

```java
public class demo {
  static class Student{
    String sname;
    int age;
    int rollno;
    String college;
  }
  public static void main(String[] args) {
    Student s1; // reference to object
    s1=new Student(); // heap allocation
    s1.sname="meow";
    s1.age=21;
    s1.rollno=123;

    Student s2=new Student();
    s2.sname="kitty";
    s2.age=22;
    s2.rollno=124;
    System.out.println(s1.sname+" "+s1.age+" "+s1.rollno); // meow 21 123
    System.out.println(s2.sname+" "+s2.age+" "+s2.rollno); // kitty 22 124
  }
}
```
This class are non-primitive data types -> can have use-defined data type
here, `new` is dynamic memory allocation at run-time

## Size of object
###### As object is like-data-type so has size for every instance
###### Reference variable(i.e is the name of the object) stored in stack takes 4/8 bytes(pointer).
```java
  class Student{
    String name;   // 4
    int age;       // 4
    int rollno;    // 4
    String college;// 4
}
```
###### here, total size of properties = 16 bytes
###### object size = 
- header size(objector header -> stores metadata(data about data)-> mark words(locks,sync,garbage) 4 bytes,class reference 4/8 bytes)
- exact field(properties-> here 16 bytes)
- padding -> optional as CPU prefer data in chunk of multiple of 8 bytes so if less than that add padding around (e.g 28 bytes -> 32 bytes so 4 bytes of padding)
example :
```java
 class person{
    bytes age;       // 1
}
```
size = (4+8 header) + 1 = 13+3(padding) == 16 bytes

###### difference between copy of object(deep copy) and copy of reference(shallow copy)
```java
public class demo {
  static void addten(pair p){
    p.x+=10;
    p.y+=10;
  }
  public static void main(String[] args) {
    pair p=new pair(10,20);
    pair p1=p;           // shallow copy
    pair p2=new pair(p); // deep copy
    p1.print();  // 10 20
    p2.print();  // 10 20
    addten(p);
    p1.print();  // 20 30
    p2.print();  // 10 20
  }
}
class pair{
    int x;
    int y;
    pair(int x,int y){
      this.x=x;
      this.y=y;
    }
    pair(pair p){
      this(p.x,p.y);
    }

    void print(){
      System.out.println(x+" "+y);
    }
}
```

## Final
###### give const variable lock it.

> [!NOTE]
> The declare and defination of final should be done in togther, but can declare it and then only once can have value in it as const

```java
public class demo {
  public static void main(String[] args) {
    shape c1=new shape(12);
    // c1.pi=19;   // ERROR: cannot assign a value to final variable pi
    c1.area();     // area of circle is 452.15999999999997
    c1.perimeter();// perimeter of circle is 75.36
  }
}

class shape{
  static String name;
  int r;
  final double pi=3.14;
  static{
    name="circle";
  }
  shape(int r){
    this.r=r;
  }
  void area(){
    System.out.println("area of "+name+" is "+pi*r*r);
  }
  void perimeter(){
    System.out.println("perimeter of "+name+" is "+2*pi*r);
  }
}
```
