###### Class inside class. it is of 4 types
- static nested
- inner class
- local class
- anonymous class
###### why nested class
- logical grouping -> class has meaning only if inside other
- better access to outer class(access private members)
## Static nested class
```java
public class demo {
  public static void main(String[] args) {
      outer.Innerdemo inner=new outer.Innerdemo();
      inner.print(); // Java
  }
}

class outer{
    private int x=5;
    static int y;
    static class Innerdemo {
        void print(){
            System.out.println("Java");
        }
    }
}
```
- this is static `class` so no need to make object of outer to access it
- it can only access static method/variables of outer class
- class can be instantiated like normal class
- can access non-static members by having a reference of outer class
```java
public class demo {
  public static void main(String[] args) {
      outer.Innerdemo inner=new outer.Innerdemo();
      outer a=new outer();
      inner.print(a); // Java 5  10
  }
}
class outer{
    private int x=5;
    static int y=10;
    static class Innerdemo {
        void print(outer o) {
            System.out.println("Java "+o.x+ "  "+y);
        }
    }
}
```
- static class can do anything that other class can do
```java
class BankAccount{
    private static class IntersetCalculator{ // helper class for only outer class
        static double calculateYearly(double p,double r){
            return p*r/100;
        }
        static double calculateMonthly(double p,double r){
            return p*r/12;
        }
    }
    public  void computeInterest(double p,double r){
        System.out.println(IntersetCalculator.calculateYearly(p,0.09));
    }
}
```
uses
- as helper for other class
- to make Builder design pattern
- If you want to have static methods inside a nested class as other types don't allow it.
## Inner class
###### class which is not static
- it belong to outer object not class thus, need to initiate the outer to make object of inner
```java
public class demo {
  public static void main(String[] args) {
      // Outer o=new Outer();
      // Outer.Inner in=o.new Inner();
      // OR WAY
      Outer.Inner in= new Outer().new Inner();
      System.out.println(in.x);   // 10
  }
}

class Outer{
    class Inner{
        int x=10;
    }
}
```
- `now outer.new Inner()` . is going inside outer object and make new Inner() 
- this inner class make new memory block in heap as it has `this` as it's reference(inner class always has reference to outer class)
- can access anything of the outer class (private /public/ static ...)
- every object of `outer` class has a inner class(because not static)
```java
public class demo {
  public static void main(String[] args) {
      Outer o=new Outer();
      // Outer.Inner in=o.new Inner();
      // OR
      Outer.Inner in= new Outer().new Inner();
      in.m1();  // 10    20
  }
}

class Outer{
    int x=20;
    class Inner{
        int x=10;
        void m1(){
            System.out.println(x + "  " + this.x);
        }
    }
}
```
- inner class can't have static methods and variable up to java-16 after it it is possible
```java
public class demo {
  public static void main(String[] args) {
      Outer o=new Outer();
      // Outer.Inner in=o.new Inner();

```

## Local class
###### class made inside a code block of `{}` ,`if`, `for`,`while`
```java
public class demo {
  public static void main(String[] args) {
      Outer o=new Outer();
      o.greet();// Hello
                // Hi
                // Bye
  }
}

class Outer{
    void greet(){
        System.out.println("Hello");
        class local{
            void sayHi(){
                System.out.println("Hi");
            }
            void sayBye(){
                System.out.println("Bye");
            }
        }
        local l=new local();
        l.sayHi();
        l.sayBye();
    }
}
```
###### It depends on business logic
- same rule as inner class but an additional rule
- rule effectively final variable rule : it can access variable which are final or effectively final(means not changes in after declare)
```java
public class demo {
  public static void main(String[] args) {
      Outer o=new Outer();
      o.greet();// Hello
                // Hi
                // Bye
  }
}

class Outer{
    void greet(){
        System.out.println("Hello");
        int x=10;
        x++; // error: local variables referenced from an inner class must be final or effectively final
        class local{
            void sayHi(){
                System.out.println("Hi "+x);
            }
            void sayBye(){
                System.out.println("Bye");
            }
        }
        local l=new local();
        l.sayHi();
        l.sayBye();
    }
}
```
- why variable must to effectively final if used in local class ?
	- local class has scope till the code block 
	- if reference of local class is pass to somewhere after the code block finished and (local class object may out live the class and code block (i.e its variable ))
	- thus, compiler will copy all variable and methods inside the local class is use(as variable can be change in code block which will lead to contradiction on it). can't keep changing copy of it
	- pass local class object by using `Object` it is a special class in java for it.
## Anonymous class
###### class without name
```java
public class demo {
  public static void main(String[] args) {
      Person p1 = new Person();
      p1.introduce();  // Hello, i am person
      Person p2 = new Guest();
      p2.introduce(); // Hello, i am guest

      // OR WAY

      Person p3 = new Person(){
      @Override
	    void greet(){
		    System.out.println("Good morning");
	    }
        void introduce(){
		    greet();
            System.out.println("Hello, i am guest");
        }
      };
      p3.introduce(); // Good morning
				      // Hello, i am guest
	  // p3.greet(); // error as not in Person class
  }
}

class Person{
    void introduce(){
        System.out.println("Hello, i am person");
    }
}

class Guest extends Person{
    @Override
    void introduce(){
        System.out.println("Hello, i am guest");
    }
}
```
###### anonymous class is child of the parent class, used for special override
- can declare new methods and variable extending the base class
- can't make constructor here( as it has no name of it's own)
- rules are same as inner class, can also make local anonymous class which follows rule of local class also
