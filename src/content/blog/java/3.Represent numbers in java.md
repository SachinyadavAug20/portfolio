## What is Bias
###### to store float Java uses LEEE standard for 32 bit and 64 bit
###### It doesn't want to store negative number in the expo (as mantissa is always positive and there is sign bit for sign don't want multiple sign bit in 1 expression) thus add maximum possible negative value for that expo size range for CPU performance
###### example:
1. for floats ![[Pasted image 20260514162600.png]] in there are 8 bits for expo thus max negative is -127 to make it positive add +127
2. for double ![[Pasted image 20260514163118.png]] here expo is 11 bits bias = (2^11 -1) = 1023
###### float and double can't store certain value in binary like 0.7 ... 
###### Java has overcame this by using BigDecimal(non-primitive data type)
