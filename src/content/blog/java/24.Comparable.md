# `Comparable` interface
class which extends this interface know how to compare
promise "I know how to compare myself with another object of my type"
```java
interface Comparable<T>{
	int CompareTo(T o);
}
```

```java
import java.util.*;

public class demo {
    public static void main(String[] args) {
        Student s1 = new Student("Sachin", 18);
        Student s2 = new Student("dmwi", 29);
        List<Student> l1=new ArrayList<>();
        l1.add(new Student("dms", 19));
        l1.add(s1);
        l1.add(s2);
        System.out.println(l1); // [< dms 19 >, < Sachin 18 >, < dmwi 29 >]
        Collections.sort(l1); // 
        System.out.println(l1); // [< Sachin 18 >, < dms 19 >, < dmwi 29 >]
    }
}
class Student implements Comparable<Student>{
    private String name;
    private int marks;
    public Student(String name, int marks) {
        this.name = name;
        this.marks = marks;
    }
    @Override
    public String toString() {
        return "< " + name + " " + marks + " >";
    }
    @Override
    public int compareTo(Student o) {
        return this.marks - o.marks;
        // this.marks - other.marks
        // < 0 : this.marks,other.marks
        // = 0 : equal
        // > 0 : other.marks,this.marks
    }
}
```

## Collection Class
This a utility class which has many static method applicable on any collection(which are more optimized)
- `Collection.sort(l1);` -> sort using intro-sort O(nlogn)
- `Collection.max(l1);` -> maximum element
- `Collection.min(l1);` -> minimum element
- `Collection.fill(l,o);` -> fill list with o fully
- `Collection.reverse(l1);` -> reverse
- `Collection.shuffle(l1);` -> shuffle(shift index randomly)
- `Collection.swap(l,a,b);` -> swap the places of this object
- `Collection.binarySearch(l);` -> works correctly on sorted list to find any element in O(logn)
- `Collection.frequency(l,a);` -> no of a appears in list l
- `Collection.unmodifiableList(l);` -> return copy of list l which is immutable(read-only)
- `Collection.unmodifiableSet(l1);`
- `Collection.unmodifiableMap(l1);`
- `Collection.emptyList();` -> gives empty list which is immutable (as not want to work with null thus will pass empty list) => null pointer exception saver => similar empty set, map, list
