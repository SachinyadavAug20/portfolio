## Input
complex than C/C++ because `java` is supposed to do server language(network input) and GUI(input) => thus, console input is complex
there is a `in` in `System` class which is static object of `InputStream`
### Streams
It means flow of data. there are two streams
- `InputStream` Data flow into program
- `OutStream` Data flow out program
this two are java abstract classes -> it say to take input and output not tell how to and from where.
Java I/O is stream based -> it can read data from different sources

There is need of many OS calls for every char(slow) -> instead talk full character input
###### make it better using `Reader`(abstract class)
![[Pasted image 20260526163829.png]] it provides method to read string lines fully -> will read full chunk of buffer (java buffer) -> buffer reader is not compatible with `inputStream` so use new class `InputStreamReader`
```java
import java.io.*;

public class demo {
  public static void main(String[] args) throws IOException {
      InputStreamReader isr = new InputStreamReader(System.in); // bytes to char
      BufferedReader br=new BufferedReader(isr); // apply buffer to read chunk of data
      String name=br.readLine(); // QWERTY
      System.err.println(name);  // QWERTY
      /*
steps : 
1. sachin -> I/P
2. OS buffer (65,100,...,97)
3. System.in  recives bytes
4. InputStreamReader  -> convert stream of bytes to char
    's' 'a'...
5. BufferedReader(stream of char readed together and stored in its buffer)  -> readline()forms string from the stream of char  -> O/P
       */
  }
}
```
Imitation of buffer reader
- can only read string `br.readline()` => for int do `Integer.parseInt(String s)`
- code is complex 
Final solution is scanner class which simplify this a lot
- simplify the input process
- gives methods to read int,double,boolean etc
Scanner is in `java.util` not in `java.io`. it is a wrapper around the `input output class` 
```java
import java.util.Scanner;

public class demo {
  public static void main(String[] args){
      Scanner sc=new Scanner(System.in); // create scanner
      String name=sc.nextLine(); // QWERT
      System.out.println(name);  // QWERT
  }
}
```
It works on tokenization(divided base on space)
methods of scanner
- `next()` -> read a word/token
- `nextline()` -> read full line till `\n`
- `nextInt()` -> read int
- `nextDouble()` -> read double
...
**scan is slower that buffer reader because it does many thing**(type conversion,regex,token)
