class which object's variable and method are immutable => once made it won't change --> like student name and roll no should be immutable
Rules of immutable
- make methods as `final` so it is not mutable by extending
- mark instance variable and private and final and can be initialized by constructor
- no setter
- All primitive are immutable and make the non-primitive also immutable(`String` is immutable)  or make defensive copy in constructor/geter
code before normal: 
```java
class Student {
    int age;
    String name;
    Student(int age, String name) {
        this.age = age;
        this.name = name;
    }
    Student(int age) {
        this(age,"person");
    }
    Student(String name) {
        this(0,name);
    }
    Student() {
        this(0,"person");
    }
    void show() {
        System.out.println("age: "+age+" | name: "+name);
    }
    public int getAge() {
        return age;
    }
    public String getName() {
        return name;
    }
    public void setName(String n) {
        name=n;
    }
}
```
make it immutable 
```java
public class demo {
```

student is not purely immutable => because of get college method which is a shallow copy(pass a reference not a exact copy thus it become mutable)
--> fix 1: make College class immutable (but can't do as name changes)
--> fix 2: defensive copy in constructor and geter
```java
public class demo {
    public static void main(String[] args) {
        Student s = new Student(20,"hui",new College("NIT","Bhoomadevapura"));

        s.show();  
        // rollno: 20 | name: hui | college: NIT | Colleg location: Bhoomadevapura
        s.getCollege().name="QWERTY"; // mutable
        s.show(); 
        // rollno: 20 | name: hui | college: NIT | Colleg location: Bhoomadevapura
    }
}

// immutable
// defensive copy
class Student {
    private final int rollno;
    private final String name;
    private final College c;
    Student(int rollno, String name,College c) {
        this.rollno = rollno;
        this.name = name;
        this.c = new College(c.name,c.location); // defensive copy
    }
    void show() {
        System.out.println("rollno: "+rollno+" | name: "+name+" | college: "+c.name+" | Colleg location: "+c.location);
    }
    public int getRollNo() {
        return rollno;
    }
    public String getName() {
        return name;
    }
    public College getCollege() {
        College newc = new College(this.c.name,this.c.location); // defensive copy
        return newc;
    }
    
}
// mutable
class College {
    String name;
    String location;
    College(String name, String location) {
        this.name = name;
        this.location = location;
    }
}
```
making defensive copy avoid multiple reference of same thing(many shallow copy) which leads to immutability===> not giving access to the reference of original object to maintain immutability i.e always give deep copy not shallow copy
Benefits of Immutable object:
- In thread can avoid race condition by having immutable objects
